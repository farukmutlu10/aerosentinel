You are an expert aviation software engineer and TypeScript architect. Your task is to write a highly robust, production-ready TypeScript parser for European (ICAO/EASA) METAR and TAF strings. 

DO NOT use US standards (Statute Miles). Strictly use European standards (Meters and Knots).

### 1. TYPESCRIPT ARCHITECTURE & BEST PRACTICES
- Define strict `Enums` for Flight Categories (VFR, MVFR, IFR, LIFR) and Color Codes (#00FF00, #0000FF, #FF0000, #FF00FF).
- Use `Interfaces` to strongly type the final output object. Use optional properties (`?`) instead of `null` where data might be missing (e.g., no clouds).
- Do not write one giant function. Break the logic down into smaller, testable helper functions (e.g., `parseWind`, `parseVisibility`, `parseClouds`, `determineFlightCategory`).
- Use Named Capture Groups in your Regex patterns for readability (e.g., `/(?<direction>\d{3}|VRB)(?<speed>\d{2})(?:G(?<gust>\d{2}))?KT/`).

### 2. FLIGHT CATEGORY CRITERIA (EUROPEAN LIMITS)
Determine the category by taking the WORST (lowest) value between Visibility and Ceiling.
- VFR (Green / #00FF00): Vis > 8000m AND Ceiling > 3000ft.
- MVFR (Blue / #0000FF): Vis 5000m - 8000m OR Ceiling 1000ft - 3000ft.
- IFR (Red / #FF0000): Vis 1500m - 4999m OR Ceiling 500ft - 999ft.
- LIFR (Magenta / #FF00FF): Vis < 1500m OR Ceiling < 500ft.

### 3. CEILING DETERMINATION (CRITICAL LOGIC)
- Cloud heights are in hundreds of feet (e.g., BKN015 = 1500 ft).
- ONLY 'BKN' (Broken) and 'OVC' (Overcast) constitute a ceiling.
- 'FEW' and 'SCT' DO NOT constitute a ceiling. Ignore their heights for flight category.
- Vertical Visibility (e.g., VV002 = 200 ft) acts as an OBSCURED CEILING. Treat 'VV' strictly as a ceiling.
- If CAVOK, NSC, NCD, SKC, or CLR is present: Ceiling is infinite (undefined/>5000ft) and Visibility is 9999 (10km or more).

### 4. WEATHER PHENOMENA ALERTS
Parse the weather string and apply these color overrides, ignoring visibility if necessary:
- DANGER (IFR / #FF0000 or LIFR / #FF00FF): If the string contains "TS" (Thunderstorm), "FG" (Fog), "+SN" (Heavy Snow), "GR" (Hail), or "VA" (Volcanic Ash), forcefully downgrade the category to IFR or LIFR. 
- SAFE (VFR / #00FF00): If the string contains "-RA" (Light Rain), "-DZ" (Light Drizzle), or "VCSH" AND visibility is >= 8000m, keep the category VFR. Do not downgrade it.
- Include a human-readable translation of the weather code (e.g., "TSRA" -> "Thunderstorm with Rain").

### 5. WIND AND GUST LOGIC
Parse the wind string (e.g., 24015KT, 18020G35KT, VRB03KT).
Find the MAXIMUM value between the sustained wind speed and the gust speed (if 'G' is present).
Assign a `dangerColorCode` based on the MAX wind/gust value:
- < 16 KT = Green (#00FF00)
- 16 KT to 24 KT = Yellow (#FFFF00)
- 25 KT to 34 KT = Orange (#FFA500)
- >= 35 KT = Red (#FF0000)

### 6. EXPECTED TYPESCRIPT INTERFACES
Your code should output an object conforming exactly to this structure:

```typescript
export interface MetarData {
  stationId: string;
  rawText: string;
  flightCategory: {
    category: FlightCategory;
    colorCode: FlightCategoryColor;
    controllingFactor: "Ceiling" | "Visibility" | "Weather" | "Both" | "None";
  };
  visibility: {
    meters?: number;
    cavok: boolean;
  };
  ceiling?: {
    feet: number;
    type: "BKN" | "OVC" | "VV";
  };
  wind: {
    direction: string | "VRB";
    sustainedKt: number;
    gustKt?: number;
    dangerColorCode: string;
  };
  weatherPhenomena: Array<{
    code: string;
    humanReadable: string;
  }>;
}
